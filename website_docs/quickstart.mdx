## Install with the installer script

The fastest way to get Claworc running. The script auto-detects your environment, asks a few questions, and handles everything else.

<Tabs>
  <Tab title="Linux / macOS">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/gluk-w/claworc/main/install.sh | bash
    ```

    Or clone the repo and run it locally:

    ```bash
    git clone https://github.com/gluk-w/claworc.git
    cd claworc
    bash install.sh
    ```
  </Tab>
  <Tab title="Windows">
    Open **PowerShell** (not Command Prompt) and run:

    ```powershell
    git clone https://github.com/gluk-w/claworc.git
    cd claworc
    .\install.ps1
    ```

    If you see an execution policy error:

    ```powershell
    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
    ```
  </Tab>
</Tabs>

The installer prompts you to choose **Docker** or **Kubernetes** deployment, then configures ports, the data directory, and pulls the necessary images automatically.

<Tip>
  To upgrade an existing installation, run `install.sh` again — it detects the current deployment and offers to upgrade in place.
</Tip>

## Finish setup in the browser

Open [http://localhost:8000](http://localhost:8000) (or the URL shown by the installer). On first run, Claworc walks you through a short onboarding wizard:

<Steps>
  <Step title="Create the admin account">
    Enter a username and password for the initial admin.
  </Step>
  <Step title="Add LLM provider keys">
    Paste API keys for any LLM providers you want available, and optionally a Brave Search key.
  </Step>
  <Step title="Choose analytics preference">
    Anonymous usage analytics are opt-in by default. Toggle off if you prefer to disable them.
  </Step>
</Steps>

---

## Create your first instance

<Steps>
  <Step title="Click New instance">
    On the dashboard, click **New instance**.
  </Step>
  <Step title="Fill in the form">
    Enter a display name (e.g., `my-agent`). Optionally set CPU and memory limits and add your API key.
  </Step>
  <Step title="Click Create">
    Claworc provisions the container and establishes an SSH connection automatically. The instance appears in the dashboard with a yellow indicator while connecting.
  </Step>
  <Step title="Wait for Connected">
    Once the SSH indicator turns **green**, the instance is fully ready.
  </Step>
</Steps>

---

## Access your instance

With a connected instance, click any of these buttons from the dashboard:

| Button | What it opens |
|--------|--------------|
| **Chrome** | The agent's browser (Chrome, Chromium, or Brave depending on your image) |
| **Terminal** | A full interactive terminal via VNC |
| **Logs** | Live streaming container output |

---

## Connection states

Each instance shows a real-time SSH connection indicator:

| Indicator | State | Meaning |
|-----------|-------|---------|
| Green | Connected | SSH tunnel is active and healthy |
| Yellow | Connecting / Reconnecting | Establishing or re-establishing connection |
| Gray | Disconnected | Not yet connected or manually stopped |
| Red | Failed | Connection failed after all retries |

<Note>
  Claworc automatically reconnects if a connection drops, including after container restarts.
</Note>
